How to Stop Teams From Auto-Starting Linux (Fedora, Debian & Ubuntu)

17/10/2022

Description

Annoyingly, Microsoft Teams on Linux Fedora, RedHat, Debian, Ubuntu, and other Linux distributions does not obey/follow the autostart settings set by your desktop environment. For these reasons, to stop Teams from automatically starting on boot, you will have to edit some directories manually using terminal.

To stop Microsoft Teams from automatically starting on boot, run the command bellow from your terminal.
$ mv /home/$USER/.config/autostart/teams.desktop /home/$USER/.config/autostart/teams.desktop.rm

It's really that simple! However, this change may be reverted on some updates. To make this change more permenant, you will have to create a cron job which runs on boot, checks to see if the file /home/$USER/.config/autostart/teams.desktop exists, and delete it if it does. The script should look something like the one bellow.

if [ -f "/home/$USER/.config/autostart/teams.desktop" ]; then
if [ if "/home/$USER/.config/autostart/teams.desktop.rm" ]; then
rm /home/$USER/.config/autostart/teams.desktop.rm;
fi mv /home/$USER/.config/autostart/teams.desktop /home/$USER/.config/autostart/teams.desktop.rm;
fi
