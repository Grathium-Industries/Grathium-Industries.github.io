Windows: command not found

02/08/2022

Fixes Errors

command is not recognized as an internal or external command, operable program or batch file

Description

This stderr standard error stream is a common Windows command line issue experienced among new programers, and we are all to common with it. But what does it actually mean?

The text "command is not recognized as an internal or external command" refers to executables or .exe files. This is stating that there is no .exe file with that name in the current directory, or in your %PATH% directories. A %PATH% directory is where command prompt looks for executables or .exe files when you type in a command. The most well known %PATH% directory is the C:\Windows\System32 directory where programs such as echo and shutdown are located. Therefore, if you type in echo "Hello World!", you are actually calling the echo program from within C:\Windows\System32 with the command line argument of "Hello World!". Hence why if you delete this directory everything stops working.)

The text "or batch file" inherits all the same ideas as the executables, but also contains the idea that you can run batch files (.bat and .cmd) files without specifying the .bat extension to the file path.
e.g. helloWorld.bat can be called by helloWorld

Fix

If you are copying code from the internet

  1. Check that you are running commands for windows command prompt, cmd, dos, or batch
  2. If there is a $ at the start of the command, remove it and any leading spaces until the first alphanumeric character (as the $ character represents that you should run the following from the command line / terminal)


General Fixes

  1. Check that you have any programs you need installed
  2. Make sure your operating system is up-to date
  3. If you need to use a batch script .bat download the script first
  4. Make sure you are in the right directory path
  5. Make sure that your %PATH% environment variables contain the path to the executable

If all else fails, you may need to try a different solution.
