What is a .gitignore File

19/07/2022

Description

A .gitignore file is a file which specifies which files to ignore and include when you are using git. It is typically located in the top level working directory of your project, and includes folders such as runtime environments, build files, node_modules, and other files that are not part of the project's source code.

Example

An automatically generated .gitignore file by Rust Lang's Cargo package manager

# Generated by Cargo
# will have compiled files and executables
debug/
target/

# Remove Cargo.lock from gitignore if creating an executable, leave it for libraries
# More information here https://doc.rust-lang.org/cargo/guide/cargo-toml-vs-cargo-lock.html
Cargo.lock

# These are backup files generated by rustfmt
**/*.rs.bk

# MSVC Windows builds of rustc generate these, which store debugging information
*.pdb

Syntax

Comments
A comment can be used to annotate a .gitignore file, and while not affect the functionality. It is typically used to describe the a purpose of a line or statement.
Usage
# this is a comment
Context
# exclude the node_modules/ directory from git
node_modules/

Excluding Folders
Excluding folders is typically used to exclude runtime environments, and specialty folders such as node_modules/ and the out/ directories.
Usage
folder_name/
Context
# exclude the node_modules/ directory from git
node_modules/

Excluding Files
Excluding files is typically used to distribution files, compiled files, temp, cache, secret files, and output files as a.out.
Usage
python_file.py
Context
# excludes the a.out files from the gcc compiler
a.out

Wildcard Expressions
The use of wildcard expressions is typically used to exclude a specific filetype from git, such as JavaScript files in a TypeScript project.
Usage
*.fileType
**/*.fileType
Context
# exclude all JavaScript files in the dist/ directory
dist/*.js # exclude all .out files from git **/*.out

Include
Including in a .gitignore is typically used when you have excluded an entire directory, but want to a limited number of files from this directory.
Usage
!fileName.py
!folderName/
Context
# include the javascript files in the dist/ folder
dist/* !dist/*.js
