How to Create a Lightweight Webpage Redirect Using Only HTML

29/07/2022

Description

Web redirects are common and useful in the modern world. They let us point the user to a new or update website, webpage, or domain. In the modern world, a lot of users will navigate away from a webpage if it takes longer than 3 seconds to load. For this reason, light and efficient redirects are a must. As any time it takes the web page to load, the more users you will lose. The three most common ways to redirect a webpage is either through DNS redirects for domain redirects, and HTML and JavaScript redirects for individual webpages.
While JavaScript redirects are useful, taking full advantage of HTML redirects will improve the time a webpage takes to load significantly (compared in a ratio to Javascript redirects, HTML:JS).

How to Use

A template for the most basic HTML redirect can be found bellow

<!DOCTYPE html>
<html>
<meta http-equiv="refresh" content="0; URL=https://grathium-industries.github.io/" />
<title>Redirect</title>
You are about to be redirected...
</html>


Tag Breakdown

<!DOCTYPE html> Tells the web browser that the current webpage should be loaded as HTML. This is typically only necessary for older web browsers


<html> While this is not required, it is good practice to keep it included so all your tags have a parent or root tag


<meta http-equiv="refresh" content="0; URL=https://grathium-industries.github.io/" /> This is the main redirect tag. It specifies that the program should redirect to the value specified in the URL=[website] parameter. Other optional parameters include time=[seconds] for a x second delay between the redirect


<title>Redirect</title> Sets the tab to show the title as "Redirect", while this is not required, it may be useful for people watch their history meticulously.


You are about to be redirected... This is just some simple text which is displayed to the user, in the event that the webpage does not load. You may wish to add a link here as well incase their web browser is outdated or doesn't support HTML redirect.
